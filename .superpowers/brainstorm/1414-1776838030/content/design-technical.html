Diseno tecnico

Como se integra la terminal en el proyecto existente. Sin frameworks, sin dependencias.

Arquitectura — un solo archivo nuevo

Estructura de archivos — cambios
+ terminal.js      ← motor completo de la terminal (comandos, typing, GitHub API)
~ index.html      ← se añade el widget HTML en el hero + <script src="terminal.js">
~ style.css       ← estilos del widget (variables ya existentes, sin romper nada)
· resto de archivos  ← sin tocar

Flujo de datos

Cuando el usuario escribe "github --stats"
Usuario escribe comando
  ↓
terminal.js parsea el input → busca en el mapa de comandos
  ↓
si es "github": fetch('https://api.github.com/users/francescguillematarrabal')
  ↓
respuesta JSON → extrae repos, lenguajes
  ↓
typing effect imprime linea a linea en el output
  ↓
listo — sin recarga de pagina, sin auth

Sistema de comandos

terminal.js — estructura interna
const COMMANDS = {
  help: () => [ "Comandos: whoami, ls projects, skills, github --stats, contact, clear" ],
  whoami: () => [ "Francesc Guillemat · DAW · Tarragona · 18" ],
  'ls projects': () => [ "web-instituto/", "poka-iot/", "viorcam-pedidos/" ],
  skills: () => [ "HTML · CSS · JS · Java · SQL · Git · MongoDB · Spring" ],
  'github --stats': async () => await fetchGitHub(),
  contact: () => [ "email: deadmanjdf@gmail.com", "github: francescguillematarrabal" ],
  clear: () => { clearOutput(); return []; }
}

Extras incluidos en el diseno

  • ✓ Historial de comandos con flecha arriba/abajo
  • ✓ Tab-completion para los comandos disponibles
  • ✓ Typing effect configurable (velocidad ajustable)
  • ✓ Mensaje de bienvenida al cargar la pagina
  • ✓ Fallback si la API de GitHub falla (mensaje de error limpio)
  • ✓ El input del teclado va directo a la terminal sin necesidad de hacer click
  • ✓ Responsive — en movil ocupa todo el ancho
